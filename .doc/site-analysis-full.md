# Полный анализ сайта ANRO TRIP (White Anrotrip)

**Дата анализа:** 2026
**Охват:** все страницы (главная, `/corp`, `/cabinet`), desktop + mobile, шрифты, изображения, переносы, адаптивность, скорость, UX, UI, SEO, управление с клавиатуры, Lighthouse (27.02.2026), тесты по веб-стандартам.

---

## Часть 1. Общий анализ сайта (все страницы)

### 1. Шрифты

#### Положительно

- **Preconnect** для Google Fonts, `display=swap` и `media="print" onload="this.media='all'"` — снижают блокировку рендера.
- Используются **Inter** (400, 500, 600) и **Montserrat** (600, 700, 800).
- `text-rendering: optimizeLegibility`, `-webkit-font-smoothing: antialiased`.
- **Fluid typography** через `clamp()` для h1–h6 без жёстких пикселей.
- В `html`: `font-size: clamp(14px, 0.875rem + 0.25vw, 16px)` — адаптивная база.

#### Проблемы

- **Fallback:** нет явного системного fallback при недоступности Google Fonts (например, `font-family: 'Montserrat', system-ui, sans-serif`).
- **FOUT/FOIT:** при `display=swap` возможны кратковременные скачки вёрстки при подгрузке шрифтов.
- На мобильных часть заголовков с очень крупным `clamp()` (например, Hero: `2.8rem … 8rem`) может быть избыточно крупной на узких экранах.

---

### 2. Изображения

#### Положительно

- Компонент **`<Image />`** из `astro:assets` — оптимизация, lazy loading.
- Hero: **AVIF**, `quality={75}`, `loading="eager"`, `fetchpriority="high"`.
- Туры, награды, партнёры — **WebP**.
- Для фонов и карточек используется `object-cover`.
- Класс `.image-sharp` с `translateZ(0)` и `backface-visibility: hidden` для стабильного рендеринга.

#### Проблемы

- **Team:** смесь WebP, PNG, JPG — PNG/JPG тяжелее и хуже для скорости.
- **Reviews:** локальный placeholder `/images/review-placeholder.svg` — до деплоя заменить на реальные фото; сейчас не placehold.co.
- **GiftModal:** `<img src="/images/gift-card-preview.jpg">` вместо `<Image />` — без оптимизации Astro.
- Явные `width`/`height` и `sizes` для критичных изображений не проверялись — возможен layout shift.
- Hero с `scale-105` и kenburns может давать артефакты на слабых мобильных GPU.

---

### 3. Переносы строк

#### Положительно

- Используются управляемые переносы через `<br />` и `hidden sm:block` (Hero, GiftSection и др.).
- Пример: «до<br class="hidden sm:block" /> полного» — разный перенос на мобиле и десктопе.

#### Проблемы

- Длинные строки без управления переносами в About, TrustBadges, Footer.
- Нет `word-break` / `overflow-wrap` для длинных слов (URL, ФИО, реквизиты).
- В CallbackModal: «Оставьте свои контакты,<br />и мы свяжемся...» — на мобиле возможен неудобный разрыв.
- В Reviews и Team используется **truncate** без `title` — обрезанный текст без подсказки.
- В TrustBadges: `sm:whitespace-nowrap` — на очень узких экранах текст может вылезать за границы.

### 4. Адаптивность (Desktop + Mobile)

#### Положительно

- Единая сетка: `px-4 sm:px-6`, `max-w-7xl`, `container mx-auto`.
- Брейкпоинты: `sm`, `md`, `lg`, `xl`.
- Hero: `min-h-dvh` — учёт высоты viewport на мобильных.
- Header: бургер на `lg`, навигация переключается по ширине.
- Partners: marquee на desktop, grid на mobile.
- Kenburns, gradient text, breathe-анимации отключены на touch через `@media (hover: hover)`.
- Для Tourvisor: `-webkit-overflow-scrolling: touch`.
- Карусель Reviews: `snap-x snap-mandatory`.
- Мобильное меню: `fixed top-[80px] right-4 w-56`.

#### Проблемы

- Header: логотип с `text-[8px]`, `text-[7px]` — очень мелко на мобиле.
- Hero подзаголовок: `max-w-xl` — на узких экранах может быть тесно.
- TrustBadges: `grid-cols-2 sm:grid-cols-3` — на мобиле две колонки, длинные значения могут не помещаться.
- Footer: на мобиле реквизиты в `text-xs` — читаемость под вопросом.
- Нет явной проверки на очень малые ширины (320px и меньше).
- Modal/CookieBanner с `max-h-[90vh]` — на малых экранах с виртуальной клавиатурой возможны проблемы.

### 5. Скорость

#### Положительно

- Astro — статический рендер, минимум клиентского JS.
- Подключены sitemap и compressor.
- Lazy loading для большинства изображений.
- Hero: `loading="eager"`, `fetchpriority="high"`.
- Учтён `prefers-reduced-motion` — анимации отключаются.
- Blur и тяжёлые анимации только при hover — экономия на мобиле.

#### Проблемы

- Шрифты с Google Fonts — дополнительный DNS и запросы.
- Tourvisor/Nemo — тяжёлые виджеты/iframe, основной риск для LCP/TTI.
- View Transitions (ClientRouter) — дополнительный JS.
- Hero: AVIF без явного fallback для старых браузеров.
- Reviews: внешние изображения — лишние запросы.
- Не проверялись: preload для LCP-картинки, размер бандла, количество запросов.

### 6. UX

#### Положительно

- Якорные ссылки: `#search`, `#about`, `#reviews` и др.
- `scroll-margin-top` для учёта фиксированного header.
- Плавный скролл к секциям (`behavior: 'smooth'`).
- Мобильное меню закрывается по клику вне области.
- При открытых модалках блокируется скролл с компенсацией ширины скроллбара.
- Есть `aria-label`, `aria-expanded` на кнопках.
- Focus visible: `focus-visible:ring-2`.
- Touch targets в основном ≥ 44px.
- Учтён `prefers-reduced-motion`.

#### Проблемы

- Логотип: сложное поведение (fade-out + навигация) — может быть неочевидным.
- PopularTours: клик по карточке без явного действия (модалка/переход) — неочевидно для пользователя.
- Reviews: `openLightbox(index)` через глобальную функцию в `onclick` — хрупко.
- CookieBanner: поведение и доступность не проверялись.
- Ссылки «Политика конфиденциальности» / «Пользовательское соглашение» ведут на `#` — заглушки.
- FavoritesWidget, OfficeWidget: инициализация по `astro:page-load` — нужно убедиться, что работают после навигации.
- GiftModal: вызов `openGiftModal()` — проверить наличие и вызов.

### 9. Управление с клавиатуры (Keyboard control)

#### Положительно

- Есть `aria-label`, `aria-expanded` на кнопках (Header, corp Hero, Footer).
- `focus-visible:ring-2` на TrustBadges — видимый фокус при навигации Tab.
- Поля форм (CallbackModal, GiftModal, ReviewModal, Reviews, cabinet) с `focus:ring`, `focus:border-primary` — фокус заметен.
- Header: кнопка меню с `aria-label="Меню"`, `aria-expanded` переключается при открытии/закрытии.
- Скролл-стрелки Hero/corp: `aria-label="Скролл вниз"` / `"Прокрутить вниз"`.
- Соцсети в Footer: `aria-label` на ссылках (VK, WhatsApp, Telegram).
- Табы SearchWidget: семантически кнопки — доступны по Tab; переключение по клику (без клавиш — см. проблемы).

#### Проблемы

- ~~**Нет Skip Link**~~ ✅ Добавлено в Layout.astro — «Перейти к содержимому» видна при фокусе.
- **Модальные окна (CallbackModal, GiftModal, ReviewModal, lightbox в Reviews):**
  - Escape для закрытия — не проверялось, возможно отсутствует.
  - Focus trap — при открытии фокус должен уходить внутрь модалки и не выходить за её пределы; при закрытии — возврат к элементу, открывшему модалку.
  - Фокус на первом интерактивном элементе при открытии.
- **Табы SearchWidget:** нет навигации стрелками (← →) и Home/End; нужны `role="tablist"`, `role="tab"`, `role="tabpanel"`, `aria-selected`, `tabindex` для roving tabindex.
- ~~**Lightbox Reviews:** Escape, стрелки~~ ✅ Реализовано (Escape, стрелки, focus trap, возврат фокуса).
- **Карусель Reviews:** горизонтальный скролл — Tab даёт доступ к карточкам, но навигация стрелками внутри карусели не проверялась.
- **Бургер-меню Header:** Enter/Space активируют кнопку (нативно), но закрытие по Escape — не проверялось.
- **PopularTours:** карточки — `<button>`; Enter/Space работают, но фокус на карточках и переход к модалке при нажатии Enter — не проверялось.
- **Однотипность `focus-visible`:** TrustBadges имеет `focus-visible:ring-2`, у многих кнопок и ссылок только `focus:outline-none` — видимый фокус может пропадать; лучше везде использовать `focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2`.
- **tabindex="-1"** на интерактивных элементах, которые должны быть вне Tab-порядка (например, декоративные) — не проверялось; `tabindex="0"` на кастомных виджетах при необходимости.

## Часть 2.5. Анализ страницы Cabinet (`/cabinet`)

Страница входа в личный кабинет туриста. Форма отправляется на внешний хост `lk.anrotrip.ru`. Виджеты (ScrollProgress, ScrollToTop, FavoritesWidget, OfficeWidget) отключены.

### 2.5.1. SEO

#### Положительно

- Указаны `title` и `description` в Layout.
- Единственный H1: «Личный кабинет туриста».

#### Проблемы

- Нет `robots noindex` — страница входа может индексироваться; для форм логина обычно используют `noindex, nofollow`.
- Нет canonical для `/cabinet`.
- Изображение: `alt=""` — допустимо для декоративного фона, но можно добавить описательный alt для доступности.

### 2.5.2. UX

#### Положительно

- Один чёткий сценарий: вход в ЛК.
- Форма компактная, `max-w-[400px]`.
- Радио Телефон/Email — переключение подсказки для поля ввода.
- `autocomplete="username"` на поле ввода.

#### Проблемы

- Нет Header — нет навигации и возврата на главную без кнопки «Назад».
- Пустой `placeholder=""` — при пустом поле неочевидно, что вводить (телефон или email).
- `input` всегда `type="text"` — при выборе «Телефон» предпочтителен `type="tel"`, при «Email» — `type="email"`.
- Нет ссылок «Забыли пароль?» / «Регистрация» — может быть намеренно, если это внешний ЛК.
- `main class="fixed inset-0"` — страница не скроллится; на мобиле с клавиатурой форма может уезжать вверх или перекрываться.
- Блок внизу (© + ссылка moidokumenti.ru) — на мобиле может перекрываться или плохо читаться.
- Отправка на внешний URL — нет индикатора загрузки и явного уведомления о переходе.

### 2.5.3. UI

#### Положительно

- Карточка формы: `bg-white/85`, тень — читаемость на фоне.
- Подпись внизу: `text-shadow` — читаемость на фоне фото.
- Кнопка «Войти» с достаточным размером.

#### Проблемы

- Нет `focus-visible:ring-2` на кнопке «Войти» — единый видимый фокус для интерактивных элементов.
- Ссылка moidokumenti.ru идёт сразу после © в том же блоке — на мобиле кликабельная область может быть неудобной.

### 2.5.4. Управление с клавиатуры

#### Положительно

- Поля формы доступны по Tab; radio, input и кнопка — в логичном порядке.
- `autocomplete="username"`.

#### Проблемы

- Радио-группа без `<fieldset>` и `<legend>` — хуже для скринридеров.
- Нет `aria-label` у radio-группы и поля ввода.
- Кнопка «Войти» без явного `focus-visible` — фокус может быть незаметен.

### 6.6. Где проводить тесты: IDE, браузер или прод

Универсального «всё в IDE» нет — часть тестов требует браузер или задеплоенный сайт.

#### Из IDE (терминал / расширения)

| Тест | Как запустить |
|------|----------------|

| **HTML Validator** | Расширение VSCode (например, «HTML validation») или CLI `vnu` |
| **axe-core** | Через npm + Playwright/Puppeteer — запуск из терминала |

#### Через браузер

| Тест | Почему |
|------|--------|
| **Lighthouse в DevTools** | Встроен в Chrome |
| **axe DevTools** | Расширение Chrome |
| **Проверка контраста** | DevTools → Inspect → Contrast |
| **WAVE** | Онлайн-сервис, нужен URL (localhost или прод) |

#### Только после деплоя (прод)

| Тест | Почему |
|------|--------|
| **Яндекс.Вебмастер** | Добавление сайта, проверка индексации, регион |
| **PageSpeed Insights** | Ориентирован на реальный URL и сеть |
| **WebPageTest** | Тест из разных регионов и устройств |
| **Rich Results Test** | Работает по URL (после добавления Schema.org) |

#### Кратко

- **IDE:** Lighthouse CLI, HTML/CSS Validator, axe через скрипты.
- **Браузер:** Lighthouse DevTools, axe, контраст, WAVE.
- **Прод:** Вебмастер, PageSpeed, WebPageTest, Rich Results.

## Часть 7. Качество визуала (анти-ИИ-шаблон)

По результатам аудита по статье [«Сайты от ИИ выглядят как сайты от ИИ»](https://habr.com/ru/articles/1004960/) (полный отчёт: `.doc/audit-ai-design-habr.md`) сайт в целом не выглядит как типичный ИИ-шаблон: есть брендовые цвета (@theme), парные шрифты (Montserrat + Inter), цветные тени и стиль Liquid Glass. Ниже — пункты, требующие доработки для соответствия рекомендациям статьи.

### 7.1. Соответствие критериям (кратко)

| Критерий | Статус | Действие |
|----------|--------|----------|
| Цвета только из @theme, без indigo/blue/sky/emerald по умолчанию | ⚠️ | Hero (градиенты по времени суток), Footer (соцсети), кнопки в модалках — перевести на токены |
| Тени многослойные, с оттенком бренда | ⚠️ | Убрать оставшиеся `shadow-md`; заменить на `shadow-xl shadow-primary/15` и аналоги |
| Два шрифта (заголовки + тело) | ✅ | Без изменений |
| Body line-height 1.7 | ⚠️ | Задать в `global.css` в @layer base для body |
| Анимации: только transform и opacity, не `transition-all` | ❌ | Заменить `transition-all` на явные свойства по всему проекту |
| Hover / focus-visible / active у кликабельных | ✅ | Без изменений |
| Система глубины (base / elevated / floating) | ⚠️ | Описать в конституции или гайде; в коде частично есть |

### 7.2. Задачи по приоритету (включены в п. 5.2)

- **Важно (по мере рефакторинга):** п. 23 — замена `transition-all`; п. 24 — цвета только @theme; п. 25 — тени.
- **Желательно:** п. 26 — body line-height; при желании — формализация уровней глубины, опционально grain в hero.

Подробные обоснования и ссылки на файлы — в `.doc/audit-ai-design-habr.md`.
