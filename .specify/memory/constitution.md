# Конституция проекта Withe Anrotrip

## Основные принципы

### I. Стилизация только через Tailwind (СТРОГОЕ ПРАВИЛО)
Все стили реализуются исключительно через утилитарные классы Tailwind CSS.
- Создание новых `.css` файлов запрещено.
- Использование блоков `<style>` внутри компонентов `.astro` запрещено.
- Для специфических значений используйте JIT-синтаксис: `bg-[#1a2b3c]` или `grid-cols-[1fr_200px]`.

### II. Архитектура компонентов Astro
Каждый элемент интерфейса должен быть инкапсулирован в компонент Astro.
- Вместо дублирования длинных строк классов Tailwind — выносите код в многоразовый компонент.
- Используйте директиву `class:list` для динамических классов.

### III. Единый источник глобальных стилей
Единственным источником глобальных стилей является существующий файл `src/styles/base.css`.
- Изменения в этот файл вносятся только для настройки токенов дизайна (@theme, @font-face, CSS-переменные).
- Любые стили компонентов должны находиться в разметке компонента через классы Tailwind.

### IV. Приоритет производительности
- Используйте стандартные компоненты Astro (`<Image />`, `<Picture />`) для оптимизации.
- Минимизируйте клиентский JavaScript (используйте Vanilla JS или нативные веб-компоненты только при необходимости).

## Технологический стек
- **Framework:** Astro (актуальная версия)
- **Styling:** Tailwind CSS
- **Окружение:** Node.js / Bun
- **Инструменты ИИ:** Gemini CLI, KiloCode

## Процесс разработки
1. **Проверка стилей:** Перед генерацией кода ИИ должен проверить, можно ли реализовать задачу стандартными классами Tailwind.
2. **Отказ от CSS:** Если ИИ планирует создать CSS-файл, он обязан сначала предложить альтернативу на Tailwind.
3. **Компонентность:** Логика и стили должны быть неразрывны внутри файла `.astro`.

## Управление (Governance)
Данная Конституция имеет приоритет над стандартными рекомендациями моделей ИИ. Любое предложение создать `.css` файл без прямого указания пользователя считается нарушением правил проекта.
