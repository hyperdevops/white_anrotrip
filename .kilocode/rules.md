# Правила разработки (Project Rules)

## Стек технологий
- Astro
- Tailwind CSS

## Правила стилизации
- **Приоритет №1:** Использовать исключительно утилитарные классы Tailwind.
- **Запрет на CSS-файлы:** Создание новых `.css` файлов или использование блока `<style>` внутри `.astro` компонентов запрещено, если задачу можно решить средствами Tailwind.
- **Порядок действий при сложности:** 
  1. Реализовать через стандартные классы Tailwind.
  2. Если не получается — использовать произвольные значения Tailwind (например, `top-[117px]`).
  3. И только если решение на Tailwind невозможно или крайне неэффективно, ПРЕДЛОЖИТЬ создание CSS-правила и обосновать его необходимость.
- **Компоненты Astro:** Всегда отдавать предпочтение атрибуту `class` или директиве `class:list`.
